## 2. 目标边界（认知层）

认知层聚焦：

- **Role**：让模型知道“我是谁、当前目标是什么”
- **Memory**：让模型记住“对后续有复用价值的信息”
- **Context**：让模型拿到“足够且不过量”的上下文
- **Validation**：让执行结果“可验证、可解释、可回放”

非目标：

- 一次性重写全部工具链
- 引入重型分布式调度体系
- 在无业务价值前提下增加复杂策略引擎

---

## 3. 下一阶段行动项

### P0（优先）

1. **单一 prompt 构建源**  
   - 目标：减少规则双写与漂移  
   - 验收：同一类规则只在一处维护，渲染层不再重复拼装

2. **能力路由与交付语义统一**  
   - 目标：主会话、子任务、渠道回传在成功/失败语义上一致  
   - 验收：关键路径都能返回结构化 envelope 且用户可见结果一致

3. **记忆写入质量治理**  
   - 目标：减少重复与低价值记忆  
   - 验收：新增记忆具备来源标识，重复率下降

### P1（增强）

1. **压缩与会话延续策略说明化**  
   - 验收：开发文档中明确两类摘要的适用边界

2. **关键路径验证分层**  
   - 验收：配置可用 -> 单轮工具 -> 多轮稳定 -> 渠道投递四层回归可执行

3. **可观测补强**  
   - 验收：关键失败场景可通过日志快速定位到阶段与原因
